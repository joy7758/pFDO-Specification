# 数字代谢熵控制一页评审材料（CN）

## 问题
园区系统长期运行后，常见痛点不是“有没有告警”，而是“是否可解释、可复现、可审计”。  
单点告警难以形成治理闭环，领导层难以判断趋势，工程侧难以验证动作效果。

## 我们的方法
我们采用 NSE-EC-1.0 最小标准骨架，用 **H(t)** 统一表达系统无序度，并用四算子（Ingest / Verify / Bind / Decay）形成可执行治理动作。  
核心口径：
- `compliance_score` 越高越好（治理质量）
- `risk_score` 越高越危险（风险暴露）
- `H(t)` 越低越有序（治理有效）

## 可复现证据
复现命令：
```bash
bash scripts/run_entropy_experiment.sh
```

代表图（仓库可直接审阅）：
- `docs/paper/figures/entropy_evolution.png`
- `docs/paper/figures/compliance_vs_entropy.png`

图示结论（评审口径）：
1. 熵曲线可区分模式与算子开关差异，具备策略比较价值。  
2. 合规与熵呈可解释关联，支撑“治理动作 -> 指标变化”的证据链。

## 价值
1. **对管理层**：从“事件响应”升级到“趋势治理”，可快速判断系统是否可控。  
2. **对工程侧**：从“经验调参”升级到“可复现实验”，便于审计与回归。  
3. **对标准化**：形成统一数据协议与解释口径，便于跨系统对接和评审复核。
