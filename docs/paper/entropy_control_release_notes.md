# 数字代谢熵控制（Digital Metabolism Entropy Control）发布说明

## 这是什么（给非技术读者）
这是一套“让系统变得更有序、更可控”的机制。
- **合规指数（越高越好）**：代表系统越稳定、越符合规则。
- **风险温度（越高越危险）**：代表系统越接近失控或事故边缘。
- **系统熵 H(t)**：代表“混乱度/不可预测性”。熵上升 = 更乱、更难管；熵下降 = 更稳、更可控。

## 你能拿它做什么（可落地）
1) **把风险变成可解释的趋势**：不是一句“有风险”，而是给出“为何上升、何时拐点、接下来会怎样”  
2) **把治理变成可验证的动作**：开启/关闭“代谢算子”，对比熵曲线变化  
3) **把演示变成可复现证据**：同一模式/同一参数，输出可重复的结果（适合评审与标准讨论）

## 核心入口（复现方法）
- 启动大屏：`bash scripts/run_park.sh` → 打开 `http://127.0.0.1:8000/park`
- 健康检查：`bash scripts/healthcheck.sh`
- 实验复现：`bash scripts/run_entropy_experiment.sh`
